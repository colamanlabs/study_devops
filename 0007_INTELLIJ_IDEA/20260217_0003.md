# Junie 시스템 프롬프트 설정

### 1. $USER_HOME/.junie/guidelines.md
```
Junie 가 사용하는 공통 시스템 프롬프트 파일
```
### 2. $APP_HOME/.junie/guidelines.md
```
Junie 가 사용하는 해당 프로젝트용 시스템 프롬프트 파일
```

### 3. 우선순위
```aiignore
공통 설정을 기본으로해서, 프로젝트 설정을 병합해서 사용한다.

......
Junie 설정 파일 구조 및 우선순위 안내
질문하신 내용에 대해 다음과 같이 답변해 드립니다.
1.
설정 파일의 역할:
◦
$HOME\.junie\guidelines.md (글로벌 설정): 사용자의 홈 디렉토리에 위치하며, 어떤 프로젝트에서든 공통적으로 적용되는 기본 시스템 프롬프트 및 코딩 규칙을 정의합니다.
◦
$APP_HOME\.junie\guidelines.md (프로젝트 전용 설정): 개별 프로젝트의 루트 디렉토리에 위치하며, 해당 프로젝트의 특성에 맞는 구체적인 규칙이나 기술 스택 관련 지침을 정의합니다.
2.
병합 및 우선순위 동작 방식:
◦
Junie는 글로벌 설정(1)을 기본으로 하되, **프로젝트 전용 설정(2)이 있을 경우 이를 병합(Merge)**하여 동작합니다.
◦
만약 두 파일의 내용이 충돌하거나 중복될 경우, 프로젝트 전용 설정(2)의 내용이 글로벌 설정(1)보다 높은 우선순위를 가집니다. 이를 통해 전체적인 기본 스타일은 유지하면서 프로젝트별로 유연하게 설정을 덮어쓰거나 추가할 수 있습니다.
따라서 프로젝트별로 특별한 규칙(예: 특정 라이브러리 사용 금지, 프로젝트 전용 명명 규칙 등)이 필요한 경우, 해당 프로젝트의 .junie\guidelines.md 파일에 정의하여 사용하시면 됩니다.
```

### 4. guidelines.md 파일 만들기
```aiignore
1. md 파일로 생성한다.

2. 지정해야 할 것들에 대한 요건을 적는다. "-" 문자로 구분해서 넣는다.

3. Junie 에게 내용을 구체적으로 해달라고 요청하면 파일 내용을 구체화 해준다.
```
